# Security Policy

This repository primarily contains a specification, schema, examples, and validation tooling. Even so, please report vulnerabilities responsibly if you identify issues in the validation scripts, GitHub Actions workflow, or any published artifacts.

## Reporting a Vulnerability

- Do not open a public issue for a suspected security problem.
- Contact the maintainers privately through GitHub with a clear description, impact, and reproduction steps.
- Include affected files, versions, and any mitigation ideas if available.

## Response Expectations

Maintainers will review the report, assess severity, and coordinate any necessary fixes before public disclosure when practical.
